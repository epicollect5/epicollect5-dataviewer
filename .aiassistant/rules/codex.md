# Codex Rules

## Execution

- Make minimal, scoped changes
- Do NOT refactor unrelated code
- Do NOT rename variables unless required
- Do NOT move files unless explicitly asked

## Code Changes

- Prefer existing patterns to new abstractions
- Prefer duplication to premature abstraction
- Do not introduce new dependencies

## Workflow

- Search before modifying code
- Read related files before editing
- Do not assume architecture — inspect it

## Output

- Keep explanations short
- Keep diffs minimal
- Do not generate unused code

## Tools

- Use grep, not rg (ripgrep is not installed)
